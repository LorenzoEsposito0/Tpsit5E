Glossario

Socket

Un socket è un endpoint per la comunicazione tra due macchine. Rappresenta l'astrazione fondamentale utilizzata per la comunicazione in rete.

SOCK_STREAM

Rappresenta un socket orientato alla connessione, utilizzato per implementare protocolli come TCP. Garantisce un flusso continuo e affidabile di dati.

SOCK_DATAGRAM

Rappresenta un socket senza connessione, utilizzato per protocolli come UDP. I dati vengono inviati in blocchi (datagrammi) senza garanzia di ordine o affidabilità.

Listen

È un'operazione che mette un socket in modalità di ascolto, permettendogli di accettare connessioni in arrivo. Utilizzato solo per socket server.

Bind

Associa un socket a un indirizzo IP e una porta specifici. È necessario per un server per specificare su quale porta accettare le connessioni.

Connect

Avvia una connessione a un socket remoto specificando l'indirizzo IP e la porta. È utilizzato dal client per connettersi al server.

Accept

Consente a un socket server di accettare una connessione in arrivo. Restituisce un nuovo socket che rappresenta la connessione con il client.

TCP

Il Transmission Control Protocol (TCP) è un protocollo orientato alla connessione che garantisce la consegna affidabile e ordinata dei dati tra due dispositivi in rete.

UDP

Il User Datagram Protocol (UDP) è un protocollo senza connessione che consente di inviare dati rapidamente, senza garantire l'ordine o l'affidabilità dei pacchetti.

Client

Il client è un dispositivo o un'applicazione che avvia una richiesta di connessione verso un server per accedere a risorse o servizi specifici.

Server

Il server è un dispositivo o un'applicazione che ascolta le richieste dei client e fornisce risposte o servizi in base alle loro richieste.
